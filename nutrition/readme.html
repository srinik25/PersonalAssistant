Nutrition Project

Automated plant-based nutrition profile generator, publisher, and notifier

Project Overview

This project automatically generates detailed nutrition profiles for plant-based foods using GPT-4o, validates the data, produces styled HTML pages, publishes them to a live website, and sends SMS notifications. It runs daily via cron.

Live site: https://wintry-tower-54cj.here.now/

Core Files

File Purpose
nutrition_generator.py Core module with 4 functions: load_config() reads config files, query_llm() calls GPT-4o for nutrition JSON, validate_data() checks structure/numbers/units + LLM cross-check, generate_html() converts JSON to styled HTML
daily_run.py Active daily runner. Generates next food, updates index.html, publishes to here.now, sends SMS. Falls back to email if SMS fails. Runs via crontab at 10:03 AM.
daily_nutrition.py Original daily runner. Picks next food from list, generates profile, and emails it. Superseded by daily_run.py.
run_batch.py One-time batch script. Ran all remaining items at once with publish + SMS after each. Already used.
send_sms.py Simple standalone SMS sender via Gmail-to-SMS gateway (email to number@tmomail.net).

Config Files

File Purpose
keys.txt OpenAI API key (GPT-4o) and Gmail SMTP credentials (sender email, app password, recipients)
system_prompt.json Defines the LLM's role as a nutrition expert and guidelines for generating accurate data
profile.json User dietary preferences (vegetarian/vegan) used to tailor nutrition advice and recipes
output.json JSON schema and example that tells the LLM exactly what format to return (fields, structure)
daily_tracker.json Tracks progress: current day index and history of all processed items with timestamps

How It Works (End-to-End)

  1. daily_run.py runs via crontab at 10:03 AM daily
  2. Reads daily_tracker.json to find the next food item in the queue
  3. Calls nutrition_generator.py which hits GPT-4o twice — once to generate nutrition data, once to cross-validate it
  4. Saves the styled HTML to output/ and regenerates index.html with the new card
  5. Publishes the entire output/ folder to here.now (permanent, authenticated)
  6. Sends SMS to both phone numbers via Gmail-to-SMS gateway; if SMS fails, falls back to email

Notifications

SMS (Primary)

Sent via email-to-SMS gateway (number@tmomail.net for T-Mobile). Both recipients get a text with the new food profile link.

Email (Fallback)

If SMS fails (e.g., carrier block), an HTML email with the profile link is sent to the Yahoo email addresses configured in keys.txt.
