How This Was Built

A nutrition website built entirely through conversation with an AI coding assistant (Vibe Coding).

Bottom Line Up Front (BLUF)

This is a nutrition website built primarily using Claude Code — Anthropic's AI coding assistant. Content is mostly AI-generated via GPT-4o, but auto-validated through cross-checks and tightly constrained by system prompts and output schemas. No code was written manually — every feature was built through plain English conversation.

Why We Built This

  • To be a useful nutrition reference site that we and friends can actually use day-to-day
  • To experiment with emerging AI capabilities — to see how far you can get building a real app purely through conversation, without writing a single line of code manually

The entire site — 40+ food profiles, search, tabs, images, recipes, Firebase, SMS, daily automation — was built in roughly 4-6 hours of conversation spread across a few sessions.

Prerequisites

  • Claude Code subscription — Anthropic's CLI tool (docs). $20/month for the base plan; $100/month for heavier Claude Code usage.
  • An LLM API key — GPT-4o (OpenAI) in this case, for content generation and validation. Estimated total API cost for this entire site: ~$8 (as of March 19, 2026) (~650 API calls including vision-based image validation).

What is Claude Code?

Claude Code is Anthropic's command-line interface (CLI) tool that lets you build software by describing what you want in plain English. It reads your files, writes code, runs commands, and iterates — all from your terminal.

Think of it as a senior developer sitting next to you, except it never sleeps and types really fast.

Installing Claude Code

Mac (via Homebrew)

# Install Claude Code
brew install --cask claude-code

# Verify it worked
claude --version

# Start coding
cd your-project-folder
claude

Mac (alternative: npm)

brew install node
npm install -g @anthropic-ai/claude-code
claude

Windows

# Install Node.js from https://nodejs.org, then:
npm install -g @anthropic-ai/claude-code
claude

After Install

  • Authenticate: Run claude in a project directory and sign in via browser with your Claude.ai account
  • Initialize: Run /init inside Claude Code to generate a CLAUDE.md — this gives Claude persistent context about your project
  • Subscription required: Claude Code only works with a paid Claude.ai account (Pro at $20/month). The free plan does not include Claude Code.

Good to Know

  • Homebrew doesn't auto-update — run brew upgrade claude-code periodically for latest features. The native installer (curl -fsSL https://claude.ai/install.sh | bash) auto-updates if you prefer.
  • Disk cleanup: Homebrew keeps old versions after upgrades. Run brew cleanup claude-code periodically to reclaim space.

Why does the npm method need Node.js? Claude Code's CLI is built as a Node.js package. Node.js is only needed to install and run Claude Code — we don't use it anywhere in the actual website (which is pure HTML/CSS/JS). npm (Node Package Manager) comes bundled with Node.js — think of it as an app store for developer tools.

Installing Python & Libraries via Claude Code

You don't need to memorize pip commands. Just ask:

# In Claude Code, just type:
"install python and the requests library"
"I need the openai library for API calls"
"set up a virtual environment and install dependencies"

Claude Code figures out the right commands (brew install python, pip install requests, etc.) and runs them for you.

Note: The backend of this site is all Python — content generation, validation, image downloading, SMS, and publishing are all Python scripts. It could just as easily have been TypeScript, Go, or any other language — Claude Code works with whatever you choose.

How It Was Built

The entire site was built through conversation. No code was written manually.

Describe what you want → Claude Code writes the code → Review & iterate → Deploy

Content Generation

  • GPT-4o generates nutrition data as structured JSON (via OpenAI API)
  • A second GPT-4o call cross-validates the data for accuracy
  • Python converts JSON to styled HTML pages

Image Sourcing

  • Images sourced from Wikimedia Commons (free, CC-licensed)
  • Each image validated by GPT-4o Vision to ensure it shows the actual food (not a plant, painting, or unrelated item)
  • Rejects and retries automatically until a clean photo is found

Features Added Through Conversation

  • Search bar with real-time filtering
  • Category tabs (Nuts, Fruits, Vegetables, Grains, Other, Recipes)
  • Browser text-to-speech ("Listen to this Content")
  • YouTube links for each food
  • Recipe submission with photo upload (the only non-AI-generated feature — actual tried-and-tested recipes added the old-fashioned way via an HTML form)
  • SMS notifications via email-to-SMS gateway (for site owners — notifies when new content is published)
  • Daily automation via macOS launchd (used for initial bulk content generation; now content is added on-demand via Claude Code, e.g., "add acai to the site")

Version Control

Local Git repo tracks all changes. Claude Code handles commits automatically — you just tell it to commit when you're ready:

# You say:
"commit the changes"

# Claude Code runs:
git add -A
git commit -m "Add new foods, Firebase recipes, image validation..."

Claude Code writes descriptive commit messages summarizing what changed, stages the right files, and excludes sensitive data. Sensitive files are excluded via .gitignore: API keys, credentials, logs.

Database Persistence

Unlike the nutrition profiles (which are static HTML files generated by Claude Code), recipes are created by users through the web frontend and need a database to persist across devices and sessions. We use Firebase Firestore (Google's free NoSQL database):

  • Reads from any device — recipes sync across laptop, phone, any browser
  • Writes are password-protected — adding, editing, and deleting recipes requires an admin password to prevent unauthorized changes
  • Drafts stay local — localStorage for work-in-progress, Firestore for published
