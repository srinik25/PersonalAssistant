Web Deployment Guide

How the Nourish & Know site is hosted, deployed, and maintained

Hosting Platform: here.now

What is it? here.now is a free static hosting service that lets you publish files and folders to the web instantly. It gives you a live, shareable URL with no server setup needed.

How We Got a Permanent Site

  1. First published anonymously (no account) — got a temporary URL that expired in 24 hours
  2. Requested a sign-in code via the here.now auth API: 
    curl -sS https://here.now/api/auth/agent/request-code \
  -H "content-type: application/json" \
  -d '{"email": "srinikatta24@gmail.com"}'
  3. Received a one-time code via email and verified it: 
    curl -sS https://here.now/api/auth/agent/verify-code \
  -H "content-type: application/json" \
  -d '{"email":"srinikatta24@gmail.com","code":"JQH4-H7JQ"}'
  4. Saved the returned API key to ~/.herenow/credentials: 
    mkdir -p ~/.herenow
echo "YOUR_API_KEY" > ~/.herenow/credentials
chmod 600 ~/.herenow/credentials
  5. Republished — the site became permanent and saved to the account

How Deployment Works

The site is published using a shell script provided by the here.now skill:

bash ~/.claude/skills/here-now/scripts/publish.sh ./output \
  --slug wintry-tower-54cj \
  --client claude-code

What this does:

  1. Create/Update — Registers the publish with the here.now API, specifying the site slug
  2. Upload files — Uploads all files from the output/ directory. Only changed files are uploaded (unchanged files are skipped for speed)
  3. Finalize — Marks the publish as complete. The site is not live until this step succeeds

The --slug wintry-tower-54cj flag updates the existing site instead of creating a new one. Authentication is read automatically from ~/.herenow/credentials.

Daily Automated Deployment

A macOS launchd agent runs every day at 7:00 AM:

~/Library/LaunchAgents/com.nutrition.daily.plist

It executes daily_run.py which:

  1. Generates the next food profile using GPT-4o
  2. Validates the nutrition data with a cross-check
  3. Saves the HTML to the output/ folder
  4. Regenerates index.html with the new food card
  5. Publishes to here.now (using the saved API key)
  6. Sends SMS via Gmail-to-SMS gateway (T-Mobile: number@tmomail.net)
  7. Falls back to email if SMS fails

Manual Republishing

To republish manually after making changes:

# 1. Regenerate the index (if foods changed)
python3 -c "
import sys; sys.path.insert(0, '.')
from daily_run import update_index_html
update_index_html()
"

# 2. Publish to here.now
bash ~/.claude/skills/here-now/scripts/publish.sh ./output \
  --slug wintry-tower-54cj \
  --client claude-code
